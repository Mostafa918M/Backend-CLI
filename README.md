# Express Backend CLI

A simple CLI tool to quickly generate Express.js backend projects and CRUD resources with a clean, scalable architecture.

This CLI helps you:

Initialize a new Express backend project

Generate resources (controller, service, routes, model)

Keep a consistent folder structure and naming conventions

Save time on repetitive boilerplate code

## Installation
### 1. Clone the CLI Repository
``` bash
git clone https://github.com/Mostafa918M/Backend-CLI.git
```

### 2. Go to the CLI Folder
```bash
cd Backend-CLI
```
### 3. Install Dependencies
```bash
npm i
```
### 4. Link the CLI Globally
```bash
npm link
```

## Usage

### Generate a New Express Project
```bash
cname g new project-name
```


This will create a fully configured Express backend project with the default structure.
you have to Install Dependencies after generate new project
```bash
npm i
```

### Generate a New Resource (CRUD)
```bash
cname g resource resource-name
```
Generates:

Controller

Service

Route

Validator (if applicable)

With Model
```bash
cname g resource name -m
```
### or
```bash
cname g resource name --model
```


Adds a model file to the resource.

### Generate a Controller
```bash
cname g controller name
```
Creates a controller without importing the service.

### With Service Import
```bash
cname g controller name --import
```
Creates a controller with service import.

### Generate a Route
```bash
cname g route name
```
Creates a route file without importing controller or validation.

## With Imports
```bash
cname g route name --import
```
Creates a route file with controller and validation imports.
